Diagnóstico de funcionamiento de una Red LAN empleando Comandos de Red TCP/IP


Con el ánimo de familiarizarse con los comandos de red TCP/IP, hemos llevado a cabo este segundo laboratorio, en cual investigamos y reconocimos los diferentes tipos de comandos de red más utilizados tanto en sistemas operativos Windows, como en Kernels como Linux (cualquier distribución), además de que obtuvimos ciertos resultados particulares que dieron pie a una serie de conclusiones como que, la destreza en el uso de comandos de red debe ser usaba con precaución y con mucho conocimiento, puesto que brindan información interna de los protocolos de red que está usando el router o AP presentes en el entorno.

Procedimiento - parte 1

Hoy en día en el mundo de la informática y de las telecomunicaciones se tienen presentes una serie de comandos TCP/IP muy comunes por consola como lo son; "ipconfig", "ipconfig /all", "ping", "tracert", "pathping", "telnet", "net","route", "arp", "nbtstat", "netsh", "getmac", entre otros.

Ahora debemos conocer cuál es la función de cada uno de esos comandos, para así garantizar el conocimiento del lector:


1. Ipconfig: Es un comando usado en Sistemas Operativos Windows, el cual tiene como objetivo mostrar por pantalla la configuración de red que está usando el ordenador en el momento, es decir determina la configuración del host (Buchanan, 2004).


2. Ipconfig /all: Al igual que en el comando Ipconfig, en este caso debemos tener presente la terminación "/all", dado que esta permite expandir la información de la red del equipo, dejando visibles aspectos como, el DNS, la IPv6, la dirección física MAC, entre otros.


3. Ping: Facilita realizar diagnósticos a un host local o remoto, el cual tiene como objetivo verificar el estado de un dispositivo interconectado (20 comandos de red más importantes en Windows, 2020).


4. Tracert: Es una herramienta de diagnóstico de red que determina una ruta especifíca a un destino mediante el envío de solicitudes eco del protocolo ICMP (Trabelsi et Al, 2006).


5. Pathping: Este comando proporciona información sobre la latencia de red y la pérdida en saltos intermedios entre un origen y un destino (20 comandos de red más importantes en Windows, 2020).


6. Telnet:Permite el control remoto de los ordenadores por medio de entradas y salidas basadas en texto. Con este objetivo, se crea una conexión cliente-servidor a través del protocolo TCP y del puerto 23, donde el dispositivo controlado ejerce de servidor y espera a los comandos pertinentes (Davidson, et Al, 1977).


7. Net: Se usa para establecer la configuración de directiva en el equipo local, como directivas de cuenta y directivas de contraseña. Este comando no se puede usar en el controlador de dominio. Este comando solo se usa en el equipo local (Deland-Han, 2021).


8. Route: Telnet permite el control remoto de los ordenadores por medio de entradas y salidas basadas en texto (20 comandos de red más importantes en Windows, 2020).


9. ARP: Muestra y modifica las tablas de conversión de direcciones IP en direcciones físicas que utiliza el protocolo de resolución de direcciones ARP (20 comandos de red más importantes en Windows, 2020).


10. Nbstat: Muestra las estadísticas del protocolo NetBIOS sobre TCP/IP (NetBT), las tablas de nombres NetBIOS del equipo local y de los equipos remotos, y la caché de nombres NetBIOS. Este comando también permite actualizar la caché de nombres NetBIOS y los nombres registrados en el Servicio de nombres de Internet de Windows (WINS) (JasonGerend, 2021).


11. Netsh: Entre las principales opciones que se pueden realizar, están la posibilidad de ver, modificar, administrar y diagnosticar la configuración de una red (Raut, 2020)


12. Getmac: Cómo su nombre indica, este comando muestra por pantalla la dirección física o MAC de la tarjeta de red del dispositivo que se esté consultando.


Procedimiento - parte 1.2 - Profundización


En esta sección vamos a profundizar temas adicionales a la investigación anteriormente realizada, con el fin de ir más allá de lo solicitado ampliando el aprendizaje base, para esto veremos lo siguiente:


1.2.1. NetBIOS: Es una API que pueden utilizar los programas en una LAN. En definitiva, se trata de un protocolo que, situado por encima de otro de más bajo nivel, permite establecer sesiones entre diferentes equipos (Glosario informático - Definición de términos informáticos, 2022).


1.2.2. Protocolo ICMP: Internet Control Message Protocol o Protocolo de Mensajes de Control de Internet, es considerado como parte de la capa de red IP, es un protocolo empleado por los routers y por los hosts para comunicar la información de control o de error de la red (Segui, 2012).


1.2.3. WINS: Windows Internet Naming Service (WINS) es un servidor de nombres de Microsoft para NetBIOS, que mantiene una tabla con la correspondencia entre direcciones IP y nombres NetBIOS de ordenadores. Esta lista permite localizar rápidamente a otro ordenador de la red (JasonGerend, 2022).

C. Un administrador de red digita el comando de red mostrado en la Figura 3. Explique el significado de: (1) bytes=32, (2) tiempo=7ms, y (3) TTL=64.



En este caso desglosaremos la respuesta parte por parte, tenemos que al ejecutar el comando "ping" en la consola de Windows, esta nos mostrará el mensaje de (en este caso), "Haciendo Ping a 192.168.1.2 con 32 bytes de datos: Respuesta desde 192.168.1.2: bytes 32 tiempo: 7ms TTL=64..." así hasta que la prueba termine su proceso de monitoreo de la red, pero ahora, ¿qué significa esto?. Bien, como bien sabemos, el comando ping testea o hace un diagnóstico a un host local o remoto, por ende se subentiende que lo primero que tenemos que conocer sí o sí, es la dirección IP a la que le vamos a hacer el monitoreo, en este caso se representa como: 192.168.1.2, luego de eso tenemos un valor importante que son los bytes de la red, en este caso son 32 bytes, esto hace referencia a la dirección IP puesto que esta dirección es un número de 32 Bytes EN IPv4 que debe ser único para cada servidor o computador presente en la red, además de que tenemos presente el tiempo en milisegundos (ms), el cual determina qué tantos X de milisegundos ha tardado la información en viajar del servidor al computador y viceversa, por lo cual nos sobra identificar el "TTL" (más visto en el ámbito personal como una "latencia"), esta sigla hace referencia al tiempo de vida del paquete enviado, donde su valor óptimo es 128, pero este puede oscilar.


Para concluir veremos una pequeña "estructura": Haciendo ping a dirección_ip con 32 bytes de datos:

Respuesta desde 192.168.1.2(IP) bytes=32(Número de bytes en IPv4) tiempo=7ms(Tiempo que tarda el paquete en ir al servidor y volver al computador y viceversa) TTL=64(Tiempo de vida del paquete envíado).


D. Un administrador de red digita el comando de red mostrado en la Figura 4. ¿Qué direcciones IP y físicas se muestran en la figura? ¿Qué significa direccionamiento dinámico?



Las direcciones que se muestran en pantalla son las direcciones IP y MAC correspondientes a cada computador conectado en la red, recordemos que el protocolo ARP tiene como finalidad de permitir que un dispositivo conectado a una red pueda obtener una ruta MAC de otro equipo que está conectado a la misma red, esto mediante la vinculación de una dirección MAC con una dirección IP, así que según está lógica, este comando está brindando la información de la dirección MAC de la tarjeta de red del ordenador y de la dirección IP con la que el protocolo ARP la está vinculando, ahora ¿qué significa el estado dinámico?, significa que estamos hablando directamente del protocolo DHCP, el cual nos permite fijar una IP que le servirá al equipo para poder navegar en internet.


E. Un administrador de red digita el comando de red mostrado en la Figura 5. Explique los términos Destino de red, Máscara de red, Puerta de acceso, Interfaz, Métrica.



Como sabemos ya, el comando netstat tiene como finalidad, mostrar por pantalla un listado de todas las conexiones de red que nuestra máquina ha realizado, pero ahora ¿cómo lo podemos identificar?. Partamos de explicar el destino de la red, que como su nombre indica, es solamente la dirección IP de destino, IP donde puede estár alojada alguna información, etcétera, por consiguiente tenemos a la máscara de subred, que para términos coloquiales cumple la función de decirle al ordenador a qué red pertenece y en qué red está trabajando, pero para fines más profesionales, la máscara de subred indica al sistema cuál es el esquema de particionamiento de subred. Esta máscara de bits está formada por la parte de la dirección de red y la parte de la dirección de subred de la dirección Internet. Por otro lado y para finalizar, tenemos a la puerta de acceso, a la interfaz y a la métrica, donde la puerta de acceso es la dirección IP que el switch o router brinda al ordenador, la interfaz es la dirección IP pública que está usando el ordenador para navegar y la métrica es el costo asociado al uso de esa ruta, es decir, es un protocolo de enrutamiento.

Procedimiento - Finalizando Parte 2.1


En está nueva sección veremos muy brevemente cuales son los comandos de red más usados en Linux (cualquier distribución):


1. Comando sudo: Abreviatura de «SuperUser Do» (SuperUsuario hace), este comando te permite realizar tareas que requieren permisos administrativos o raíz. Sin embargo, no es aconsejable usar este comando para el uso diario, ya que podría ser fácil que ocurra un error si haces algo mal (Deyimar A, 2017).


2. Comando ping: Usa el comando ping para verificar tu estado de conectividad a un servidor. Por ejemplo, simplemente ingresando ping google.com, el comando verificará si puedes conectarte a Google y también medirá el tiempo de respuesta (Deyimar A, 2017).


3. Comando Useradd, userdel: Dado que Linux es un sistema multiusuario, esto significa que más de una persona puede interactuar con el mismo sistema al mismo tiempo. useradd se usa para crear un nuevo usuario, mientras que passwd agrega una contraseña a la cuenta de ese usuario. Para agregar una nueva persona llamada John, escribe useradd John y luego para agregar su contraseña, escribe passwd 123456789 (Deyimar A, 2017).
